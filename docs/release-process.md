# Release Process

This repo uses small, readable releases.

## Checklist

1. Run `npm run check`.
2. Confirm CI passes on `main`.
3. Update `CHANGELOG.md` if the release includes user-facing changes.
4. Create a GitHub release with:
   - highlights
   - how to run the demo
   - website link
   - next roadmap focus

## Version Style

- `v0.1.x`: polish, docs, small fixes.
- `v0.2.x`: retrieval quality and eval reporting.
- `v0.3.x`: provider adapters and browser UI.
